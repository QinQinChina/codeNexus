# packages/app/src/renderer/components

## 目录用途

渲染层应用壳组件目录，负责页面骨架、聊天/时间线展示和通用 UI 组件。

完整功能工作台组件优先由对应 feature 包导出，app 本地组件只保留壳层和通用展示能力。

## 当前结构

| 目录         | 说明                                       |
| ------------ | ------------------------------------------ |
| `layout/`    | 应用骨架、Settings、侧栏、聊天与编辑器布局 |
| `timeline/`  | 时间线卡片壳与卡片内容                     |
| `ui/`        | 通用交互组件                               |
| `workflow/`  | 工作流编辑、运行与画布                     |
| `guardian/`  | guardian 诊断组件                          |
| `mcp/`       | MCP 资源面板                               |
| `userInput/` | 用户输入挂载组件                           |

## 维护边界

- ✅ 组件层负责展示与轻量交互编排
- ✅ 功能包组件通过包入口接入 app，而不是复制到本目录
- ❌ 复杂业务规则优先下沉到 `features/`、`domain/`、`stores/` 或独立 feature 包
